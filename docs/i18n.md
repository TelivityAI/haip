# Internationalization (i18n)

The HAIP dashboard uses [`react-i18next`](https://react.i18next.com/). This doc
explains how it's wired and how to contribute a translation.

## How it works

- **Library:** `i18next` + `react-i18next`, initialised in
  [`apps/dashboard/src/i18n.ts`](../apps/dashboard/src/i18n.ts).
- **Source of truth:** `apps/dashboard/src/locales/en.json`. English is also the
  **fallback** — any key missing (or left as `""`) in another locale renders the
  English string, so the UI never shows a blank or a raw key.
- **Language detection:** a user's saved choice (localStorage key `haip.lang`)
  wins, otherwise the browser language, otherwise English. Users switch language
  from the selector in the header.
- **Keys are namespaced** by area: `nav.*` (sidebar), `header.*`, etc. Add new
  namespaces as new top-level objects.

Strings are read in components with the `useTranslation` hook:

```tsx
import { useTranslation } from 'react-i18next';

function Example() {
  const { t } = useTranslation();
  return <button aria-label={t('header.logout')}>{t('nav.dashboard')}</button>;
}
```

## Adding or completing a translation (e.g. German)

1. Open `apps/dashboard/src/locales/de.json`. It mirrors `en.json`'s exact key
   structure; untranslated values are empty strings `""`.
2. Fill in the German values. Keep keys identical — only translate the values.
   A few entries are pre-filled as examples (e.g. `nav.reservations`,
   `header.selectProperty`).
3. Leave a value as `""` if you're unsure — it safely falls back to English.
4. Verify locally: `pnpm --filter @telivityhaip/dashboard dev`, then use the
   header language switcher (EN/DE) to preview.

Prefer working in a **spreadsheet**? Export the `en.json` keys, translate in two
columns (key → value), and open a PR or attach the sheet — we'll import it.

## Adding a brand-new language

1. Create `apps/dashboard/src/locales/<code>.json` mirroring `en.json`.
2. In `apps/dashboard/src/i18n.ts`: import it, add it to `resources`, and add an
   entry to `SUPPORTED_LANGUAGES` (`{ code, label }`). The header switcher and
   detection pick it up automatically.

## Adding a new UI string

1. Add the key + English text to `en.json` (and the same key to every other
   locale, value `""` until translated).
2. Use `t('namespace.key')` in the component — never hardcode user-facing text.

## Notes

- Date/number formatting in the header is not yet localized (date-fns locale
  switching is a planned follow-up). Translations of static strings are
  unaffected.
- Only the header and sidebar are migrated so far as the reference pattern;
  other pages are migrated incrementally.
